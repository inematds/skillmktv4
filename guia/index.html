O que é

Um empacotador de skills de marketing

SkillMKT v4 não é um app — é um repositório com 6 pastas de skill (cada uma um SKILL.md com identidade, regras e referências) e um script que as compacta em arquivos .skill prontos pra distribuição.

📦 6 skills, 1 pasta cada

Cada skill vive em skills/<nome>/SKILL.md, com sua própria descrição de trigger e, em várias, uma pasta references/ com material de apoio.

🗜️ Build vira .skill

build.sh zipa cada pasta em dist/<nome>.skill — o formato pronto pra instalar num agente Claude que suporte skills.

🎯 Foco em tráfego pago

Criativo, página, reel, lançamento e mensageria cobrem o funil de marketing pago de ponta a ponta, do gancho do anúncio ao pitch do evento.

Como funciona

Do código-fonte da skill ao arquivo distribuível

O fluxo é simples: cada skill é uma pasta com SKILL.md (e, opcionalmente, references/); o build zipa cada pasta separadamente.

pasta skills/<nome>/→ ./build.sh→ zip por skill→ dist/<nome>.skill→ instala no agente
Pré-requisitos

O que precisa estar disponível

Não há dependências de runtime além de utilitários de linha de comando padrão.

zip

O build.sh chama zip diretamente — precisa estar instalado no PATH.

# checar se tem zip
which zip

bash

O script de build é um shell script simples, sem outras dependências.

# dar permissão de execução (se preciso)
chmod +x build.sh

Um agente com suporte a skills

Os arquivos .skill gerados são pensados pra ser instalados num agente Claude que carregue skills (ex.: Claude Code).

# nenhum comando extra — instalação depende do agente-alvo
Guia de uso · passo a passo

Gerar e usar os arquivos .skill

Três passos: clonar, buildar, instalar a skill que interessa.

1

Clone o repositório

Traga o projeto pra sua máquina.

git clone https://github.com/inematds/skillmktv4.git  # baixa skills/ e build.sh
cd skillmktv4
2

Rode o build

Gera um .skill (ZIP) por pasta dentro de skills/, salvando tudo em dist/.

./build.sh  # -> dist/criador-criativos.skill, dist/designer-senior.skill, etc.
3

Instale a skill que precisar

Cada .skill é independente — instale só as que fizerem sentido pro seu agente/fluxo (ex.: extraindo o ZIP na pasta de skills do seu Claude Code).

unzip -o dist/criador-criativos.skill -d ~/.claude/skills/  # exemplo: instala a skill de criativos
4

Ou edite direto na pasta skills/

Como cada skill é só uma pasta com SKILL.md, dá pra editar o conteúdo antes de rodar o build de novo — sem precisar descompactar nada.

${EDITOR:-nano} skills/criador-reels-c1/SKILL.md  # ajusta o prompt/instruções
./build.sh  # regenera o .skill com a mudança
As 6 skills

O que cada .skill entrega

Resumo do trigger e escopo de cada uma, direto dos respectivos SKILL.md.

🎯 criador-criativos

Criação, análise e iteração de criativos (anúncios) para tráfego pago: hooks, body, estáticos, UGC, engenharia reversa de anúncio, diagnóstico e otimização de métricas.

📄 criador-paginas-low-ticket

Páginas de vendas para tráfego frio e ofertas low-ticket: criação do zero, diagnóstico, headlines, reescrita de dobra, teste A/B — sempre com diagnóstico antes de gerar copy.

🎬 criador-reels-c1

Gera 3 roteiros de Reels de conteúdo de autoridade (C1, sem venda direta) para o nicho de Marketing e Vendas, com gancho nos 15s iniciais e CTA solto no final.

🎨 designer-senior

Inteligência de design front-end: páginas, identidade visual, paleta, tipografia, design system — com blacklist explícita de padrões visuais genéricos a evitar.

🚀 lancamento-pago-semanal

Sistema completo de lançamento pago semanal (modelo 5+1): diagnóstico, venda de ingresso, página, criativos, tráfego, estrutura do evento e pitch.

💬 mensageria-lancamento

Mensageria completa de eventos pagos semanais (WhatsApp/ManyChat): onboarding, antecipação de aula, ao vivo, abertura de carrinho — cada mensagem com uma função só.
